Critical Apparatus & Comments

Chant transcription is rarely simple. Different manuscripts of the same chant often contain variant spellings, different pitches, or entire melodic revisions. Monodi-Zero includes a powerful **Critical Apparatus** system to document and analyze these parallel readings inline.

The Comment Tree Model

Unlike standard plain-text comments, Monodi-Zero represents annotations as a **recursive comment tree**. This structure allows you to build complex comparisons. Hover over the nodes below to preview their visual structures in the sidebar:

Node Type Purpose & Behavior
Undecided Node The initial state of a newly created comment segment. It serves as a placeholder where you choose whether to make the node a simple Leaf (content) or a comparative Grid (variants).
Leaf Node A node containing concrete content. Leaves support three kinds of content:
  • Text: Plain text or markdown for explanations, literature citations, or notes.
  • Notes: Alternate musical notes/syllables transcribed from variant sources.
  • Bracket: Grouping containers.
Grid Node A spreadsheet-like matrix layout. It aligns parallel variants:
  • Rows represent different variant manuscripts.
  • Columns represent sequential melodic segments.
Because the tree is recursive, each cell in a Grid is itself another Comment Tree (which can be Leaf, Undecided, or even nested Grids).

Creating a Comment (Two-Step Selection)

Comments are anchored to a precise range of notes in the transcription. Creation is a guided, explicit process so the range is never ambiguous:

  1. Start: Open the Comments tab in the document sidebar and click Add Comment. A banner appears above the editor showing the two steps.
  2. Pick the start note: Click the note where the annotated passage begins.
  3. Pick the end note: Click the note where it ends. The range may span syllables and even staff lines; start and end are ordered automatically, and the comment editor opens for the selected range.

You can cancel at any point via the banner. Existing comments are drawn as brackets beneath the notes they cover. When a note carries several comments, the brackets are colour-coded and the matching comment cards in the sidebar use the same colours, so every bracket can be identified at a glance. Comments are edited or deleted from the sidebar cards or by clicking a bracket.

Interacting with the Editor

Use the live interactive panel on the right (visible when not hovering over the triggers above) to practice building a critical comment tree:

  1. Create a Node: The panel starts with a blue Undecided node. Click on Leaf or Grid.
  2. Choose Leaf Type: If you chose Leaf, click Text or Notes to instantiate the editor. Click on the text box to enter comments or notes.
  3. Build a Grid Matrix: If you chose Grid, you will see a cell. Click the small circular Action Dot () adjacent to cells to:
    • Insert parallel columns (variants over time).
    • Insert parallel rows (different manuscript witnesses).
    • Configure source labels (e.g., click an action dot on the left of a row and choose Set Context to name a row after a specific manuscript shelfmark like CH-E 121).

Scholarly Print Rendering

When you export the document to PDF, the rendering engine reads the grid alignments and formats them horizontally directly under the primary chant line, matching standard critical musicology editions. Text annotations are compiled as numbered footnotes, keeping the sheet clean and easy to read.
