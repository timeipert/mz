Settings & Customization

Monodi-Zero is highly customizable, allowing researchers to adapt it to their projects' specific metadata schemes, liturgy structures, print layouts, and XML schemas.

Configuration Tabs

Click on Settings in the main navigation bar to configure the following areas:

1. Metadata Schemas

Each musicological project has different indexing requirements. Monodi-Zero lets you customize the database fields:

  • Predefined Values: Add dropdown options for standard fields (like Origin, Order, Feast Day, or Genre) to ensure consistent spelling and cataloging among different transcribers.
  • Custom Fields: Need to store something else (like Composer, Scribe, or a Cantus ID)? Click New Custom Field, choose whether it applies to the parent Source or individual child Documents, and name it. It will instantly appear in the metadata panel.

2. GitHub Synchronization

Configure the credentials to back up and share your transcriptions:

  • Connection parameters: Supply your GitHub username, repository name, branch (e.g. main), and a Personal Access Token (PAT).
  • Ensure your PAT is generated with repo access.

3. PDF Export & layout

Tune visual details before exporting to PDF. Adjustments are instantly applied to the print rendering engine:

  • Scaling & Margins: Set the global page scale, apparatus staff sizes, and page margins in points (pt).
  • Staff & Spacing Indents: Set the indent on the left of staves to leave space for manuscript sigla. Adjust vertical spacing between staves and lyrics offsets.
  • Lemma Brackets: Fine-tune the thickness, height, and offsets of brackets that group critical apparatus segments.
  • Typography: Choose between Serif (Times) and Sans-Serif (Helvetica) fonts, and adjust font sizes for titles, metadata, running headers, and apparatus comments.
  • Synopsis PDF: Options for the synoptic-comparison export — staff scale, plus toggles for the title block, the witness table, the date, and the footer with sigla and page numbers.

4. Dynamic Container Names

Chants have different poetic and structural levels (e.g., Strophes, Verses, or Sections). To keep your transcription editor aligned with these structural divisions, you can map liturgical Genre combinations (Genre 1 & Genre 2) to custom level labels:

  • For an Antiphon, you might set the levels to: Level 1 = Antiphon, Level 2 = Verse, Level 3 = Syllable.
  • For a Sequence, you might set: Level 1 = Double Strophe, Level 2 = Versicle, Level 3 = Segment.
  • The editor will automatically update its structural labels to match your profile mappings.

5. MEI XML Export Profiles

If you need to load your Monodi-Zero transcriptions into external rendering suites (such as Verovio or MEI databases), you can fully customize the generated XML structure using the graphical MEI mapping editor.

Note for scholars: The default profile targets the MEI neume module (MEI 5.0). Existing exports are unchanged unless the profile is edited.

  • Profile Management: Create, duplicate, rename, or delete mapping profiles. You can also export a profile as JSON to share with colleagues or import an external profile. The "active profile" determines the output of your exports.
  • Entity Tree: The left column displays the structural hierarchy of Monodi (from Document Skeleton down to Notes and Paratext). You can toggle the "enable" checkbox on any entity to include or exclude it from the final XML output.
  • Rule Editor: Select an entity in the tree to edit its rules in the middle column. Here you can define the XML tag, manage nested wrappers around the entity, and bind XML attributes to static strings or dynamic data fields using the bounded field vocabulary. You can also dictate where the element's inner text content is pulled from.
  • Document Skeleton: At the root of the tree, you can define the outermost wrapping tags for the entire exported XML document (e.g. <mei> and <music>).
  • Live Validated Preview: The right column renders a live, formatted preview of your current configuration using a sample document. The system instantly validates your rules, providing amber warnings for empty tags, duplicate attributes, or invalid field bindings, and red alerts if the configuration generates invalid XML structure.

6. HTML Edition Export

Configure the self-contained offline edition viewer generated by Export HTML edition (see Data & Workspace):

  • Frontpage, header & footer: Provide your own HTML snippets for the edition's landing banner, the running page header, and the footer (e.g. copyright and licensing notes).
  • Custom CSS: Inject project-specific styling into the generated site.
  • Metadata selection: Choose which source and document metadata fields are displayed in the published edition.

7. Workspace

The Workspace tab is the control centre for your locally stored data:

  • Storage usage: Compute counts of sources, documents, and transcription rows, see the browser storage quota, and detect orphaned records.
  • Import / Export: Import corpora from ZIP archives (monodi and OMMR4all layouts), import/export full workspace backups (JSON), export a monodi-compatible ZIP, or generate the HTML edition.
  • Cleanup & Danger Zone: Remove orphaned records, reset settings or UI preferences, delete all documents, or wipe the entire workspace — each guarded by a two-click confirmation.

The complete guide to data handling is on the Data & Workspace page.

Undo & Redo System

To protect your transcription data, Monodi-Zero tracks every edit, insertion, and structural split in an in-memory **Undo/Redo Stack**:

  • History Size: Configure how many steps are kept in history (default is 15) under Editor settings.
  • Scope: History covers pitch changes, note insertions, line splits, clef adjustments, syllable spelling edits, and comment updates.
  • Keyboard Shortcuts:
    Action Windows/Linux macOS
    Undo Ctrl + Z Cmd + Z
    Redo Ctrl + Shift + Z Cmd + Shift + Z
