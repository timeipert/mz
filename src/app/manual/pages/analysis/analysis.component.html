Analysis & Statistics

Beyond transcription and search, Monodi-Zero includes three analytical tools that work across your whole corpus: the Synoptic Comparison for viewing parallel witnesses side by side, the Melodic Formula Analysis for discovering recurring melodic patterns, and the Notation Statistics page for quantitative pattern distribution across sources.

Synoptic Comparison

The synopsis aligns two or more transcriptions of the same (or related) chants so you can compare their melodies measure-for-measure — the digital equivalent of a synoptic edition page.

Starting a comparison

  1. Run any search on the Search page (Quick, Sources, Documents, or Melody tab).
  2. Tick the checkboxes of the documents you want to compare. A floating action bar appears at the bottom of the screen showing your selection.
  3. With two or more documents selected, click Compare. The synopsis view opens.

Alignment modes

The toolbar at the top offers four strategies for deciding which syllables line up in the same column:

  • Melody — witnesses are aligned by their melodic profile. Best when the melodies are closely related but the texts differ.
  • Text — witnesses are aligned by their lyric text. Best for textually stable chants with melodic variance.
  • Structure — alignment follows the structural containers (sections, verses, lines) you created in the editor's Structure tab. Sections that a witness lacks are shown as empty staves, with a matched n/m count per section.
  • Sequential — lines are simply paired in document order, with no content matching.

Display options

  • Consensus text — hides the per-witness lyric text and shows a single consensus row below the staves instead. Where witnesses disagree, all variant readings are listed. This makes the view considerably denser.
  • One line (Melody/Text modes) — renders each witness as one continuous horizontal strip with a scrollbar, instead of wrapping into segments.
  • Witness table — a compact metadata table above the staves listing each witness's siglum, incipit, feast, and other fields. Use the Columns button to choose which metadata columns appear.

PDF export

The Export PDF button produces a fully vector PDF: staves and neumes are true vector paths and all text is selectable. The PDF has a classic layout — a serif masthead on page one, the witness table, a running head on subsequent pages, and a footer with the witness sigla and real Page X of Y numbers. Header, footer, date, and sigla can be toggled in Settings → PDF. In One-line mode the export produces a single custom-sized page containing the whole strip.

The current comparison (selected documents, alignment mode) is encoded in the URL, so you can bookmark a synopsis or share the link with colleagues who use the same workspace.

Melodic Formula Analysis

The formula analysis scans a set of documents for recurring melodic patterns — formulas, standard phrases, or borrowed material — and groups every occurrence across the corpus. It is opened from the Search page and always analyses the documents in your current result set, so you can scope the analysis by searching first (e.g. only one genre, one feast, or one source).

Parameters

  • Type — how melodies are compared:
    • Pitch names: absolute pitches must match (transpositions are distinct patterns).
    • Intervals: sequences of intervals; transposed statements of the same formula are grouped together.
    • Contour: only the up/down/same shape matters — the loosest grouping.
  • Minimum length — the number of notes a pattern must have to be reported.
  • Strictness — exact matches only, or fuzzy to also group slightly varied statements.
  • Merge overlapping — combines patterns whose occurrences overlap into compound groups (with a configurable minimum overlap).
  • Duplicate detection — before analysing, near-identical documents (the same chant transcribed twice, e.g. after an import) are detected and all but one excluded so they do not inflate the counts. Detected duplicates are listed so you can review them.

The heavy computation runs in a background worker: the interface stays responsive, a progress bar reports each phase, and long runs can be cancelled. Results are cached — if you navigate away and back, or reload the page, the analysis is restored (recomputed automatically in the background after a reload) instead of you having to set it up again.

Reading the results

  • Overview — every document is drawn as a horizontal timeline bar; coloured segments mark where each pattern occurs. This gives an instant picture of which formulas saturate which chants. Clicking a coloured segment jumps directly to that pattern in the list view.
  • List — one card per pattern group, ordered by frequency, with the pattern rendered on a staff and every occurrence listed (witness, document, position, context). The list is paginated; cards can be expanded to show all occurrences, and each occurrence links straight into the editor.

Every pattern card has a permalink (#pattern-group-N in the URL). Bookmarking or sharing that URL reopens the analysis on the right page with the card highlighted. Analyses can also be saved as named sessions and reloaded later.

Notation Statistics

The Stats page (main navigation) provides a quantitative view of notation patterns across your entire workspace, independent of any search.

  • Pattern × Source matrix — rows are neume patterns (grouped by their base shape, with ornament signs such as quilisma, oriscus, strophicus and liquescents stripped), columns are your sources. Each cell shows how often the pattern occurs in that source, so notational habits of individual scriptoria become visible at a glance.
  • Variant breakdown — clicking a pattern name opens a modal listing its concrete variants (with ornament signs), how often each variant occurs, and in which sources.
  • Occurrence drill-down — clicking a matrix cell lists every single occurrence (document, folio, line, syllable). Clicking an occurrence opens the transcription editor at exactly that position.
  • Filters — a minimum-frequency slider, a pattern search box, and a source search box narrow the matrix; all lists are paginated.

Because computing the statistics reads every document, it only runs when you click Compute Workspace Stats. The result is cached in the browser: navigating away, reloading, or even restarting the browser shows the cached numbers instantly, with the computation date displayed. Click Recalculate whenever your transcriptions have changed.

You can also jump into the statistics from the transcription editor: the context menu of a note offers show this pattern in Stats, which opens the page pre-filtered to that pattern.
