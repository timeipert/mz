Data & Workspace

Monodi-Zero is a local-first application: your entire workspace — sources, documents, transcriptions, and settings — lives in your browser's IndexedDB storage. There is no server and no account. This page explains what that means, how to move data in and out, and how to maintain a healthy workspace.

How your data is stored

  • All data is kept in the browser on the machine you are using. Opening Monodi-Zero in a different browser or on a different computer shows a different (initially empty) workspace.
  • Transcriptions are stored as one database row per document, so even very large corpora (hundreds of sources, thousands of chants) stay fast and safe to save.
  • Because the browser owns the data, clearing site data in your browser settings deletes the workspace. Make regular backups using the export options below or GitHub synchronization.
  • Storage usage and browser quota can be inspected under Settings → Workspace.

Getting data in

All import/export actions live in Settings → Workspace.

  • Import ZIP — imports a zipped corpus folder. Two folder layouts are detected automatically:
    • the monodi export structure (as produced by Monodi's export_monodi tooling and by this app's own ZIP export), and
    • OMMR4all exports.
    Large imports (hundreds of sources) run in the background with a progress dialog showing each phase — reading, parsing, importing sources, importing documents, saving — and the interface stays responsive throughout.
  • Import workspace (JSON) — restores a complete workspace backup previously created with Export workspace. This replaces the current workspace contents.
  • Pull from GitHub — if GitHub sync is configured (see Settings), the cloud icons in the navigation bar pull the shared state from your repository.

Getting data out

  • Export workspace (JSON) — one file containing every source, document, transcription, and your settings. The recommended full-backup format.
  • Export ZIP — a monodi-compatible ZIP of the corpus, suitable for re-importing here or into other monodi tooling.
  • Export HTML edition — generates a self-contained, offline edition viewer: a static HTML site with your transcriptions rendered as notation, browsable metadata, and built-in search. Ideal for publishing an edition or handing a read-only copy to colleagues without Monodi-Zero. The frontpage, running header, footer, custom CSS, and which metadata fields appear are all configurable under Settings → HTML Export.
  • Push to GitHub — publishes the workspace state to your configured repository for backup and team sharing.
  • Per-document PDF and MEI — from the transcription editor toolbar; see Metadata & Export.
  • Synopsis PDF — from the synoptic comparison; see Analysis & Statistics.

Workspace maintenance

Settings → Workspace also contains housekeeping tools:

  • Storage usage — click Compute Workspace Stats to count sources, documents, and transcription rows, check the browser storage quota, and detect orphans. Results are cached with a timestamp; recompute on demand.
  • Orphan cleanup — over time (usually after partial imports or deletions) the database can accumulate orphan documents (whose parent source was deleted) and orphan transcription rows (whose document was deleted). One click removes them and reclaims storage without touching live data.
  • Reset app settings — restores default settings (custom fields, PDF layout, MEI mappings) while keeping all sources and documents.
  • Reset UI preferences — clears remembered column layouts and dismissed hint cards.

Danger zone

  • Delete all documents & notes — empties the corpus but keeps your curated sources and settings. Useful before re-importing chant data.
  • Delete everything — a full factory reset: sources, documents, transcriptions, settings, and preferences.

Every destructive action uses a two-click confirmation: the first click arms the button (it changes colour and asks you to confirm), the second click within a few seconds executes it. Export your workspace first if in doubt, and reload the page after a wipe so all cached state is rebuilt.
