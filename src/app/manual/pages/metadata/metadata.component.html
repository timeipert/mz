Metadata & Exporting

To support academic cataloging and interoperability, Monodi-Zero includes an advanced metadata indexing system and support for standard scholarly exports (MEI and PDF).

Document Metadata Fields

The **Metadata Panel** is located in the document sidebar. Keeping this metadata accurate is essential for project-wide searching and cataloging. Hover over the fields below to see them highlighted in the mock sidebar panel on the right:

  • Document ID: A unique identifier for the chant within your database.
  • Text Initium: The incipit or first few words of the lyrical text, used for quick piece-level searches.
  • Genre 1 & 2: Liturgical genre categories (e.g., Antiphon, Responsorium, Introitus) for catalog classification.
  • Feast / Feast Day: The liturgical feast to which this chant belongs (e.g., In Nativitate Domini).
  • Custom Fields: Project-specific fields defined in settings (e.g., database references like Cantus ID or manuscript shelfmarks).

PDF Vector Export

Monodi-Zero features a built-in rendering engine that translates the internal digital representation of staff lines, clefs, syllables, and neumes directly into a publication-quality vector PDF — no browser print dialog involved:

  1. Click the PDF Export button in the editor toolbar.
  2. The A4 sheet is drawn natively: staves and neumes as true vector paths, lyrics and metadata as selectable text, and critical-apparatus segments with lemma brackets. Everything scales without pixelation.
  3. The layout — page scale, margins, staff spacing, fonts, bracket geometry, and which metadata fields appear in the header — is fully configurable under Settings → PDF.
  4. The finished file downloads directly as Document_<ID>.pdf.

MEI (Music Encoding Initiative) Export

The **Music Encoding Initiative (MEI)** is an open, XML-based standard for encoding musical scores, widely used in digital humanities research. When you export to MEI:

  • Monodi-Zero translates the syllables, staff lines, clefs, and notes into structural XML elements like <syllable>, <neume>, and <nc> (neume component) wrapped inside <staff> and <layer> tags.
  • All metadata edited in the sidebar is automatically mapped to standard MEI header tags (<meiHead>) representing the title, genre, liturgy, and source.
  • The exported .mei XML file can be read and rendered by standard music notation tools (e.g., Verovio) or used for advanced computational corpus analysis.
