Use Cases

To help you get the most out of Monodi-Zero, here are step-by-step guides for common research workflows.

Use Case 1: Transcribing a Melody and Printing it as PDF

Imagine you have a single manuscript page that you want to transcribe, preview, and then save as a cleanly formatted PDF document for a presentation or publication.

Step 1: Create or Open a Document

First, navigate to the Sources page from the main navigation bar. Open the source that contains your manuscript and create a new document within it.

Step 2: Set the Initial Metadata

In the right-hand Metadata panel, enter the basic information:

  • Text Initium: The first few words of the chant.
  • Folio Start / Line Start: Ensure these match where the chant begins on the physical manuscript page.

Step 3: Enter the Transcription

Follow these steps to quickly encode the notation:

  1. Insert a Clef: Focus the editor, press c to insert a clef, and pick the appropriate pitch context (e.g. C-Clef on staff line 3).
  2. Add Syllables: Go to the syllable text row (bottom), type your text, and press Space to move to the next syllable.
  3. Add Notes: Use the ↑ arrow key to jump to the notes above a syllable. Press Space to insert a note, and ↑/↓ arrows to adjust the pitch.
  4. Special Note Shapes: Use l for Liquescents or o for Oriscus shapes as you transcribe.

Step 4: Export to PDF

Once the transcription looks complete:

  1. Click the PDF Export button (marked with a PDF icon) on the main top-right toolbar.
  2. A dialog will ask if you want to include the document's Metadata table. Check the box if you want a clean metadata header at the top of your PDF.
  3. Click Print / Save PDF.
  4. Your browser's native print dialog will appear. Make sure the destination is set to Save as PDF, enable background graphics, and click Save!

Use Case 2: Setting up a Collaborative Team Project

If you are working with a group of researchers or students, you need a way to share data securely and track everyone's changes. Monodi-Zero natively integrates with GitHub to act as a backend database.

Step 1: Create a GitHub Repository

  1. Create a standard repository on GitHub (e.g., my-monodi-project).
  2. Invite all your team members as collaborators to this repository.

Step 2: Configure Monodi-Zero

Each team member must configure their local Monodi-Zero instance to connect to this repository:

  1. Click on Settings in the top navigation bar.
  2. In the GitHub Synchronization section, enter the repository name (e.g., username/my-monodi-project).
  3. Enter a Personal Access Token (PAT). Each team member needs to generate their own PAT on GitHub with repo permissions.

Step 3: Collaborative Workflow

Once connected, a "Sync" indicator will appear in the top navigation bar. Here is the daily workflow for your team:

  • Start of the Day: Click the Pull button (cloud with a down arrow) to pull the latest changes that other team members made.
  • Working: Work normally. Monodi-Zero saves all changes to your local browser storage instantly, so you never lose work if you close the tab.
  • End of the Day: Click the Push button (cloud with an up arrow) to upload your daily work to GitHub.

Resolving Conflicts

Because Monodi-Zero stores each document as a separate JSON file, team members can work on different documents simultaneously without issue. If two members edit the exact same document at the same time, the last person to Push will be warned to Pull and resolve the changes first.

Use Case 3: Melodic Variant Analysis via the Critical Apparatus

As a musicologist, you often need to document spelling or melodic differences between your primary manuscript source and other concordant sources. Monodi-Zero allows you to build a structured critical apparatus directly inline.

Step 1: Select the Target Syllable or Note

Identify the syllable in your transcription editor where the variant occurs. Focus on that syllable or its notes in the editor.

Step 2: Add a Critical Comment

Open the context menu or click the **Add Comment** button. This opens the Comment Panel on the right side and inserts a new recursive comment block initialized in the **Undecided** state.

Step 3: Create a Comparative Grid

  1. In the comment tree panel, click **Grid** on the Undecided block. This turns the block into a comparative spreadsheet matrix.
  2. By default, it creates a single cell. Click the **Action Dot** (small round button next to the cells) to add columns (parallel melodic segments) or rows (alternative manuscript sources).
  3. Click on the cells inside the Grid. You can turn each cell into a **Leaf** representing:
    • Notes: Input alternative notation syllables or pitches.
    • Text: Write scholarly commentary or bibliography references (e.g., "C-clef shifts in Bamberg ms.").

Step 4: Label your Sources

Click on the **Action Dot** at the left of each row to open the Context menu. Select **Set Context** and type the name or shelfmark of the variant manuscript (e.g., Einsiedeln 121 or St. Gall 339). The row will now display that label, organizing your apparatus.

Step 5: View in PDF

Export your document to a PDF. The critical apparatus will render cleanly, aligning the variant rows horizontally and displaying your annotations directly underneath the main chant line, ready for scholarly publication.

Use Case 4: Reconstructing Damaged Folios

Liturgical manuscripts are often centuries old and suffer from physical damage (tears, ink fading, or water stains). Monodi-Zero combines image annotation with critical apparatus markup to support editorial reconstruction.

Step 1: Map the Damaged Area

Open the document in Split Screen mode. Zoom in on the damaged section of the manuscript in the IIIF viewer. Draw an annotation zone covering the illegible stave or text fragment and click Link.

Step 2: Create a Reconstruction Block

In the transcription editor, create a corresponding Line or Syllable block. Link the line to your drawn IIIF zone so that they remain logically bound together.

Step 3: Encode Hypothesized Readings

  1. Open the comment panel for the illegible syllable.
  2. Create a **Leaf** of type **Text** or **Notes**.
  3. Input your editorial hypothesis. Use standard critical brackets (e.g., [al]leluia or notes inside brackets) to represent reconstructed pitches.
  4. In the justification text, cite supporting evidence (e.g., "Reconstruction based on parallel chant in St. Gall 390, folio 42r.").

Step 4: Export with Editorial Apparat

When exporting to PDF or MEI, the reconstructed reading is cleanly tagged as an editorial hypothesis, keeping your primary transcription strictly factual while providing scholars with fully documented reconstructions.

Use Case 5: Classroom & Pedagogical Corpus Transcription

Instructors teaching paleography or medieval musicology can use Monodi-Zero to collaborate on class-wide transcription projects, tracking student work via GitHub.

Step 1: Set up the Course Repository

The instructor creates a GitHub repository (e.g., university-chant-project-2026) and commits a set of empty source documents (holding only metadata, IIIF links, and blank staffs).

Step 2: Connect Student Instances

Each student connects their browser's Monodi-Zero instance to the central repository under **Settings**, using their personal GitHub account credentials.

Step 3: Assigned Transcription Workflow

  1. Students open the app and click Pull to download the template files.
  2. Each student is assigned a specific folio range to transcribe.
  3. As students work, their progress is saved automatically to local storage. They can work offline without losing data.
  4. At the end of their session, students click Push to upload their work to the course repository.

Step 4: Central Review and Merging

Because each chant document is stored in its own separate file in the repository, students can work in parallel without merge conflicts. The instructor can open the central repository to review student transcriptions, download the completed MEI corpus for analysis, or compile a single PDF chant book containing the work of the entire class.
